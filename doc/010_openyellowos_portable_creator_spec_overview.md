# open.Yellow.os Portable Creator 仕様メモ（概要 + GUI設計）

## 1. 概要

### 1-1. 目的
本アプリケーションは、**現在動作中の Linux システムを USB メモリへ移植し、ポータブル Linux 環境を作成するツール**である。

本ツールで作成される USB は、一般的な Live USB とは異なり、以下の特徴を持つ。

- squashfs を使用しない
- overlayfs を使用しない
- persistence を使用しない
- ext4 ベースの通常の Linux システムとして動作する
- apt upgrade / package install / 設定変更が通常の Linux と同様に行える

これにより、**「自分の環境をそのまま持ち運べる USB Linux」** を実現する。

---

### 1-2. 基本コンセプト

#### ① Live USB ではなく「通常の Linux を USB に作る」
本ツールは ISO 展開ツールではない。  
現在のシステム、または既存のポータブル USB を元にして、USB 上に通常の Linux システムを構築する。

#### ② overlayfs を使わない
overlayfs は構造が複雑になりやすく、USB メモリとの相性や保守性の面でも不利なため、本設計では採用しない。

#### ③ persistence を使わない
persistence は Live 構成前提の概念であり、本ツールの「フル Linux インストール型 USB」では不要とする。

#### ④ USB メモリ向け最適化を行う
USB メモリはランダム書き込みが遅く、書き込み寿命にも限界があるため、頻繁に書き込まれる領域を RAM(tmpfs) に逃がす前提で最適化する。

例:
- `/tmp`
- `/var/tmp`
- `/var/log`
- `/var/cache`
- `~/.cache`
- ブラウザキャッシュ

#### ⑤ バックアップしやすい構成にする
作成されたポータブル USB は、USB → USB バックアップや将来的な復元機能にもつなげやすい構成とする。

---

### 1-3. 想定する主機能

#### A. 現在のシステムを USB にコピー
現在起動中の Linux システムを USB に移植し、ポータブル Linux として起動可能な状態にする。

#### B. ポータブル USB をバックアップ
既存のポータブル USB を別の USB へコピーし、バックアップ用 USB を作成する。

#### C. 将来的な拡張候補
- ポータブル USB の復元
- ポータブル USB の修復
- USB イメージの保存 / 復元
- open.Yellow.os 以外の Debian / Ubuntu 系への対応

---

### 1-4. 対象ユーザー
- open.Yellow.os ユーザー
- 自分専用の Linux 環境を持ち運びたいユーザー
- USB ポータブル Linux のバックアップを取りたいユーザー
- 開発環境 / 検証環境 / レスキュー環境を USB 化したいユーザー

---

### 1-5. 出力される USB の特徴

- BIOS / UEFI 起動に対応
- 通常の Linux と同じように更新・設定変更可能
- ログイン機能を引き継げる
- `/etc` や `/usr` を直接編集可能
- 別 PC で mount / chroot / 修復がしやすい
- USB 向け最適化を適用して、書き込み負荷を抑えられる

---

### 1-6. 既存ツールとの違い

| ツール | 主な方式 |
|---|---|
| Rufus | ISO 書き込み |
| Ventoy | ISO 起動 |
| MX Live USB | Live + persistence |
| 本ツール | 現在システム / 既存 USB をベースにしたフル Linux 移植 |

## 2. GUI設計

### 2-1. GUI設計方針
GUI は **ウィザード型** とする。

理由:
- 破壊的操作を段階的に確認できる
- コピー元 / コピー先の概念を整理しやすい
- ストレージ誤選択を防ぎやすい
- 初心者にも分かりやすい
- 将来の復元 / 修復機能も追加しやすい

---

### 2-2. 画面構成概要

#### モードA: 現在のシステムをUSBにコピー
1. ホーム / モード選択
2. コピー元確認
3. コピー先USB選択
4. オプション設定
5. 最終確認
6. 実行中
7. 完了

#### モードB: ポータブルUSBをバックアップ
1. ホーム / モード選択
2. バックアップ元選択
3. バックアップ先選択
4. オプション設定
5. 最終確認
6. 実行中
7. 完了

---

### 2-3. 画面1: ホーム / モード選択

#### 目的
ユーザーに実行したい機能を選ばせる。

#### 表示項目
- アプリ名
- 簡単な説明文
- 機能カード

#### モード候補
1. **現在のシステムをUSBにコピー**
2. **ポータブルUSBをバックアップ**
3. **ポータブルUSBを復元**（将来対応）
4. **ポータブルUSBを修復**（将来対応）

---

### 2-4. 画面2A: コピー元確認（現在のシステムをUSBにコピー）

#### 目的
現在動作中のシステムがコピー元であることを確認し、引き継ぐ内容を選ばせる。

#### 表示項目
- OS名
- ホスト名
- root デバイス
- 現在ログイン中のユーザー
- システム使用容量
- ホームディレクトリ使用容量
- 推定必要容量

#### オプション
- 現在のユーザー設定を含める
- ホームディレクトリを含める
- 他のユーザーも含める
- 保存済みWi-Fi設定を含める
- SSH鍵を含める

---

### 2-5. 画面2B: バックアップ元選択

#### 目的
既存のポータブル USB をバックアップ元として選択する。

#### 表示項目
- デバイス名
- 容量
- モデル名
- マウント状況
- open.Yellow.os ポータブル環境らしき構成の検出結果

---

### 2-6. 画面3: コピー先USB選択

#### 目的
コピー先となる USB を選択する。

#### 表示項目
- モデル名
- `/dev/sdX`
- 容量
- 接続種別
- 現在マウント中かどうか
- 内容が消去されることの明示

#### 補助機能
- デバイス一覧更新
- 内蔵ディスク表示切替（危険）
- 容量不足警告
- システムディスク選択防止

---

### 2-7. 画面4: オプション設定

#### 目的
作成・バックアップ時の動作や最適化内容を決める。

#### プリセット
- おすすめ
- USBメモリ向け
- SSD向け
- 詳細カスタム

#### 基本設定例
- BIOS / UEFI 両対応で作成する
- USB向け最適化を有効にする
- このUSB専用の識別情報を作り直す
- SSH接続用のホスト鍵を作り直す

#### 作成モード向けオプション
- ログインパスワードを引き継ぐ
- 自動ログイン設定も引き継ぐ
- キャッシュを掃除して容量を減らす
- 保存済みWi-Fi設定を含める
- SSH鍵を含める

#### 詳細設定例
- `/tmp` をRAMに保存する
- `/var/tmp` をRAMに保存する
- ログをRAM中心で扱う
- ブラウザキャッシュをRAMに保存する
- APTキャッシュもRAMに保存する
- GNOMEサムネイルキャッシュをRAMに保存する

---

### 2-8. 画面5: 最終確認

#### 目的
実行内容をユーザーに最終確認させる。

#### 表示項目
- コピー元
- コピー先
- 何が消去されるか
- 何を引き継ぐか
- どの最適化を適用するか
- 注意事項

#### 確認チェック
- 内容確認と消去同意のチェックボックス

---

### 2-9. 画面6: 実行中

#### 目的
現在の処理状況を表示する。

#### 表示項目
- 全体進捗バー
- 現在の処理
- 処理ステップ一覧
- 詳細ログ表示
- ログ保存ボタン
- 中止ボタン

#### ステップ例
1. コピー先USBを初期化しています
2. パーティションを作成しています
3. ファイルシステムを作成しています
4. システムをコピーしています
5. 起動設定を作成しています
6. USB向け最適化を適用しています
7. 最終確認を行っています

---

### 2-10. 画面7: 完了

#### 目的
処理結果をユーザーへ伝える。

#### 成功時
- 作成 / バックアップ成功メッセージ
- 作成先デバイス情報
- 次に行う操作の案内
- ログ保存ボタン
- バックアップ作成への導線

#### 失敗時
- 失敗したステップ
- エラー概要
- ログ保存案内
- 前画面へ戻る
- 閉じる

---

## 3. 共通UI仕様

### 3-1. ウィンドウ
- 単一ウィンドウ
- リサイズ可能
- 初期サイズ: 960x720 前後
- 最小サイズ: 820x620 程度

### 3-2. 共通ヘッダー
- アプリ名
- 現在のモード名
- 手順番号表示（例: 手順 2 / 5）

### 3-3. 共通フッター
- 戻る
- 次へ
- 実行
- キャンセル

### 3-4. 入力検証
- モード選択済み
- コピー元 / 先選択済み
- コピー元とコピー先が異なる
- 容量が十分
- システムディスクをコピー先にしていない
- 消去同意チェック済み

## 4. 文言設計指針

### GUIで優先する表現

| 内部用語 | GUI表示候補 |
|---|---|
| tmpfs | RAMに一時保存 |
| machine-id | このUSB専用の識別情報 |
| SSH host key | SSH接続用の鍵 |
| rootfs | システム本体 |
| system clone | 現在のシステムをコピー |
| wipe | 内容をすべて消去 |

### GUIで前面に出さない用語
- overlayfs
- persistence
- squashfs
- lowerdir
- upperdir
- whiteout

必要ならヘルプやログでのみ扱う。

## 6. まとめ
このアプリは、**ISOを書き込むツールではなく、現在の Linux 環境や既存のポータブル USB を元にして、通常の Linux と同じように扱えるポータブル USB を作るツール**として設計する。

GUIは、危険な操作を安全に進められる **ウィザード型** を採用し、

- ストレージ誤選択を防ぐ
- 引き継ぐ内容を明確にする
- USB向け最適化を自然に選ばせる
- 初心者にも分かりやすい

ことを重視する。
