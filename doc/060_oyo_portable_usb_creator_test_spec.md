# oyo-portable-system-creator

## テスト仕様書 v1

本書は **oyo-portable-system-creator**
の品質保証のためのテスト仕様を定義する。主目的は以下である。

-   USB作成機能の正確性確認
-   誤操作によるディスク破壊の防止確認
-   BIOS / UEFI 起動確認
-   USB環境での安定動作確認

------------------------------------------------------------------------

# 1. テスト方針

テストは次の4レベルで行う。

  レベル           内容
  ---------------- ------------------
  ユニットテスト   内部関数の検証
  統合テスト       ワークフロー検証
  システムテスト   USB作成機能
  実機テスト       USB起動確認

------------------------------------------------------------------------

# 2. テスト環境

## 2.1 開発環境

  項目     内容
  -------- ----------------
  OS       Debian 12 / 13
  Python   3.10以上
  GUI      PyQt6

------------------------------------------------------------------------

## 2.2 テスト用ストレージ

  用途        容量
  ----------- ----------
  USBテスト   32GB以上
  USBテスト   64GB以上

------------------------------------------------------------------------

## 2.3 テストPC

最低2種類

-   BIOS専用PC
-   UEFI PC

可能であれば

-   NVMe搭載PC
-   SATA SSD PC

------------------------------------------------------------------------

# 4. 詳細テストケース

------------------------------------------------------------------------

## T01 USBデバイス検出

### 目的

USBデバイスのみ表示されることを確認する。

### 手順

1.  USBを2本接続
2.  GUIを起動
3.  デバイス一覧を表示

### 期待結果

-   USBのみ表示される
-   NVMe / SATA は表示されない

------------------------------------------------------------------------

## T02 root disk 誤選択防止

### 目的

システムディスクが選択できないことを確認する。

### 手順

1.  root disk を確認
2.  GUIデバイス一覧表示

### 期待結果

-   root disk が表示されない
-   または選択不可

------------------------------------------------------------------------

## T03 容量不足検出

### 目的

容量不足USBを検出する。

### 手順

1.  8GB USBを接続
2.  システム容量20GB

### 期待結果

-   USB選択不可
-   容量不足表示

------------------------------------------------------------------------

## T04 パーティション作成

### 目的

正しいパーティション構成を作る。

### 期待構成

  番号   用途
  ------ -----------
  1      EFI 512MB
  2      root ext4

### 確認コマンド

    lsblk

------------------------------------------------------------------------

## T05 フォーマット

### 目的

正しいFS作成確認

### 確認

    blkid

期待

-   vfat
-   ext4

------------------------------------------------------------------------

## T06 rsyncコピー

### 目的

システムコピー確認

### 確認

    ls /mnt/root

存在

-   /etc
-   /usr
-   /home

------------------------------------------------------------------------

## T07 GRUBインストール

### 確認

    grub-install
    chroot /mnt/usb update-grub

成功ログ

------------------------------------------------------------------------

## T08 BIOS起動

### 手順

1.  USB挿入
2.  BIOS boot menu

### 期待

Linux起動

------------------------------------------------------------------------

## T09 UEFI起動

### 手順

1.  UEFI boot menu
2.  USB選択

### 期待

Linux起動

------------------------------------------------------------------------

## T10 USB最適化

### 確認

    mount | grep tmpfs

期待

-   /tmp
-   /var/tmp

------------------------------------------------------------------------

## T11 firstboot処理

### 確認

    cat /etc/machine-id

コピー元と異なる

------------------------------------------------------------------------

## T12 ログ出力

### 確認

    /tmp/oyo-portable-creator.log

存在

------------------------------------------------------------------------

## T13 非対応構成の拒否

### 目的

v1 非対応構成（LVM / mdraid / dm-crypt）で処理が開始されないことを確認する。

### 手順

1. root が LVM または暗号化の検証環境を用意
2. GUI/CLI で実行開始

### 期待

- 事前検証で停止する
- 「v1では非対応」の文言を含むエラーが表示される
- 破壊的処理（partition/format）へ進まない

------------------------------------------------------------------------

## T14 失敗時クリーンアップ

### 目的

失敗時に mount/chroot が残留しないことを確認する。

### 手順

1. rsync または grub ステップで意図的に失敗させる
2. 失敗後に mount 状態を確認

### 期待

- `/mnt/usb` 配下がアンマウントされる
- 実行ログに cleanup 実施記録が残る
- 再実行は先頭から可能

------------------------------------------------------------------------

# 5. 異常系テスト

  ID    内容        期待結果
  ----- ----------- ------------
  E01   USB抜去     エラー停止
  E02   rsync失敗   ログ出力
  E03   grub失敗    警告表示
  E04   mount失敗   処理停止

------------------------------------------------------------------------

# 6. パフォーマンステスト

測定項目

  項目          内容
  ------------- --------
  コピー速度    GB/min
  USB書込速度   MB/s
  実行時間上限  20GB→32GB USB で 120分以内
  メモリ使用量  アプリ増分 512MB 以下
  ログサイズ    1実行あたり 50MB 目安

------------------------------------------------------------------------

# 7. 回帰テスト

v2開発時

必ず確認

-   USB作成
-   BIOS起動
-   UEFI起動
-   firstboot

------------------------------------------------------------------------

# 8. v1合格条件

以下を満たす

-   必須ケース（T01〜T07, T10〜T14）の成功率 95%以上
-   BIOS起動成功（T08）
-   UEFI起動成功（T09）
-   root disk 誤選択不可
-   ログ取得可能
-   実行時間/メモリ/ログサイズの基準を満たす

補足:
- 実機依存で再現しない事象は「既知問題」として切り分け、
  再現条件・回避策・影響範囲を記録する。
